# 跨平台与客户端

> 类型：规范。核心/适配层、平台支持级别和客户端边界以本文为唯一权威来源。

## 三层边界

| 层 | 内容 |
| --- | --- |
| 跨平台核心 | 领域、规则编译、Catalog、任务、HTTP/WS、内容哈希、相对路径协议 |
| 平台适配层 | FileIdentityProvider、FileWatcher、PathCanonicalizer、ProcessController、ToolDiscovery、AppDirs、CredentialStore |
| 桌面壳专属 | DirectoryPicker、Tray、Notification、AutoStart、OpenExternal、PowerEvents、单实例 UI |

核心不得读取注册表、COM/WMI、盘符、Win32 路径、FSEvents、Keychain 或 WebView。平台文件身份只作为候选线索，不能改变 ContentBlob 的权威身份。

## 适配端口

| 端口 | 最低职责 |
| --- | --- |
| FileIdentityProvider | 返回平台身份线索、卷/设备范围和可靠性，不承诺内容相同 |
| FileWatcher | 发出可能变化事件、溢出/丢失状态和重扫提示 |
| PathCanonicalizer | 原始/比较路径、大小写和 Unicode 语义、根内复核 |
| ProcessController | 受控启动、取消、超时、退出码和资源限制 |
| ToolDiscovery | 显式配置与受控搜索、版本探测、能力报告 |
| AppDirs | 配置、数据库、日志、缓存、临时、状态和运行描述符的安全位置 |
| CredentialStore | 当前 OS 用户范围的 secret 保存、读取、吊销和不可用降级 |

适配器错误必须转换为稳定领域 code，并保留可诊断但不泄密的内部原因。

## 支持级别

不得把“可交叉编译”写成“正式支持”。平台状态统一为：

| 级别 | 含义 |
| --- | --- |
| 可构建 | CI 能生成目标二进制，仅证明依赖边界未阻塞编译 |
| 核心已验证 | 在目标系统实际运行核心探针、数据库、路径、扫描和 API |
| 发行候选 | 安装/便携、升级、权限、凭据、工具、崩溃恢复和签名流程通过 |
| 正式支持 | 进入发布矩阵，有持续 CI、故障文档和回归责任 |

## v1 最低支持矩阵

| 目标 | v1 最低要求 | 当前证据状态 |
| --- | --- | --- |
| Windows 11 x64 / NTFS | 正式支持 | 核心探针已运行；发行、junction/reparse 和最终壳门禁待完成 |
| Linux x64 / 原生 ext4 | 至少核心已验证；若提供发行包则达到发行候选 | 当前仅 WSL DrvFS 运行和 Linux 交叉编译，不能视为 ext4 通过 |
| macOS 当前支持版本 / APFS | 可构建；是否进入 v1 正式支持由门禁决定 | 当前仅交叉编译 |
| Docker Linux x64 | 发行候选后方可列入支持 | daemon、只读挂载和非 root 运行未验证 |
| SMB/NAS/UNC Source | 作为独立存储能力门禁，不随宿主 OS 自动获得支持 | 断线、重挂载和身份稳定尚未验证 |

如果时间不足，v1 可以只正式支持 Windows，但核心仍必须保持可移植，不得用 Windows 专有实现穿透领域层。

## Web/PWA

响应式 Web/PWA 是唯一业务 UI 基线：

- 普通浏览器可完成 Source 以外的全部业务流程；涉及目录选择时可输入/粘贴受控路径或由 CLI/壳协助；
- 所有数据、权限、排序和状态来自正式 API，不直读文件系统或 SQLite；
- 断线后通过 HTTP snapshot 恢复，不依赖 WebSocket 事件历史；
- 设计系统使用 headless 可访问组件 + 自建语义令牌，支持暗色/浅色和 reduced motion；
- 键盘、焦点、ARIA、对比度、触控目标和媒体查看是发布门禁；
- 浏览器本地状态只保存展示偏好，不成为业务事实权威源。

## 桌面壳边界

桌面壳只能：

```text
启动/停止或连接 galleryd
单实例与进程归属
目录选择
托盘/通知
开机自启
打开外部 URL/文件位置
CredentialStore bootstrap
电源事件
```

壳不能访问数据库、Catalog service、扫描器对象、规则执行器或绕过 API 授权。WebView 与普通浏览器加载同一 Web 应用、使用同一 Session/capability 和 OpenAPI。

## 后端进程契约

`galleryd` 独立启动并绑定 loopback 动态或配置端口；原子写入包含 protocol version、端口、PID、启动 nonce 和 ownership 的 descriptor。壳必须先验证用户权限、nonce 和健康状态，不猜测固定端口。

契约还需定义：壳启动、用户独立启动和系统服务启动的 ownership；graceful shutdown；orphan policy；后端崩溃退避；最大重启次数；协议不兼容错误；更新期间的数据版本检查。

## Wails 的状态

Wails 是当前 Windows 优先薄壳原型，状态为**有条件接受**。它已证明最小壳可构建和启动，但在 Linux/macOS GUI、托盘、自启、目录选择、CredentialStore、签名/公证、自动更新和 Tauri 对照完成前，不得冻结为不可替换产品依赖。

无论选 Wails、Tauri 或无壳模式，`galleryd`、Web UI、OpenAPI、Session 和数据层保持不变。对应决策见 [ADR-007](../ADR/ADR-007-跨平台适配层.md) 与 [ADR-008](../ADR/ADR-008-可替换桌面壳.md)。

## 更新和发行

- 程序资源与用户数据分离，覆盖更新不得删除 AppDirs；
- 数据 Schema 升级必须先备份 control，Catalog 不兼容时可重建但不能丢 Overlay；
- Web 资源与后端 API 版本一起发行；壳更新不得私自修改 API；
- Windows 签名、macOS 签名/公证、Linux 包权限和容器镜像固定依赖分别验收；
- 平台能力不可用时返回清晰降级状态，例如 CredentialStore 不可用不能静默明文保存。

详细门禁见 [测试与发布门禁](../指南/02-测试与发布门禁.md)，已有平台证据见 [验证记录](../证据/验证记录.md)。
