# 产品定义与不变量

> 类型：规范。产品范围和不可违反原则以本文为唯一权威来源。

## 产品定义

**Gallery** 是面向个人及可信小型网络的本地优先媒体目录产品。它从用户明确注册的只读媒体 Source 中提取结构化事实，建立可重建 Catalog，并通过同一套正式 API 向 Web、PWA、桌面壳和未来客户端提供浏览、搜索、媒体读取、用户组织和后台任务能力。

正式命名固定为：

| 场景 | 名称 |
| --- | --- |
| 英文公开产品名 | Gallery |
| 中文公开产品名 | 画廊 |
| 代码、仓库、包、命令和服务代号 | `gallery` |
| 后端服务建议名 | `galleryd` |
| 本地命令行建议名 | `galleryctl` |

“下一代”“Next”等词不得成为产品名或代码命名空间。

## Gallery 是什么

- 只读媒体目录、可重建索引和受控媒体服务；
- 将多个物理 Source 和多个外部 Provider 组织成统一 Canonical 视图的个人知识库；
- 支持用户覆盖、收藏、隐藏、进度、备注、分享等不可重建状态的长期产品；
- API-first 的单机服务，Web UI 和桌面壳只是同一后端的客户端；
- 默认可在 Personal 模式零服务器运维使用，并可显式启用 LAN 模式。

## Gallery 不是什么

- 不是文件管理器；v1 不改名、移动或删除原媒体文件；
- 不是下载器、抓取器或云同步系统；
- 不是外部 Provider 的业务逻辑集合，不在代码中按平台名分支；
- 不是必须依赖桌面壳才能运行的桌面应用；
- 不是默认面向公网的多租户服务；
- 不以任何既有系统的数据格式、接口或行为作为兼容目标。

## 产品模式

| 模式 | v1 状态 | 公开范围 | 身份方式 |
| --- | --- | --- | --- |
| Personal | 必须交付，默认 | loopback | 一次性配对建立管理 Session；匿名不自动成为管理员 |
| LAN | 必须交付，可选启用 | 明确配置的局域网地址 | 本地账户、Session/API Token、capability 授权 |
| Remote | 延后，默认关闭 | 反向代理后的远程访问 | HTTPS、OIDC 和受信代理模型通过独立门禁后再引入 |

## v1 产品范围

v1 必须形成一条完整闭环，而不是只实现扫描器：

- 创建 Library、注册只读 Source、绑定不可变 RuleVersion 和参数；
- 编译规则并执行 Dry Run、Explain 和 Impact；
- 扫描 Source，原子发布 Catalog revision，并可取消、重试和崩溃恢复；
- 浏览 CanonicalWork/CanonicalCreator/CanonicalMedia，按结构化条件过滤，并以 `query_publication_id` 执行正式排序和快照分页；
- 支持至少双字符 CJK 连续搜索、拉丁/文件名中缀和精确标签过滤；
- 以稳定 CanonicalMedia ID 提供 Range、ETag 和条件请求；
- 支持 Canonical Binding、至少一种 Metadata Override、Favorite 和 ReadingProgress，并证明重建 Catalog 后仍保留；
- Personal 浏览器配对、LAN 本地账户、Session/API Token、capability 和 WebSocket 吊销闭环；
- 响应式 Web/PWA；桌面壳不得成为 v1 后端和 Web 功能的前置条件。

## 延后范围

- 原媒体写操作和回收站；
- Remote/OIDC、公网托管和多租户；
- WASM 或外部进程插件运行时；
- 原生移动客户端；
- 压缩包、PDF 和漫画容器的核心内建解析；
- 单字符 CJK 大库无过滤搜索、自动拼音/日文读音、模糊纠错；
- 外部独立搜索服务；
- 自动识别或导入其他产品数据。

## 系统不变量

1. **媒体根只读**：任何数据库、日志、缓存、临时文件或派生资源都不得写入 Source 根及其子树。
2. **用户事实不可被重扫覆盖**：Binding、Override、Collection、Favorite、Progress、Note、Share 和账户状态必须独立于可重建 Catalog。
3. **路径不是身份**：相对路径是 FileLocation；CanonicalWork、CanonicalMedia 和 ContentBlob 各有不同身份语义。
4. **Catalog 只发布完整 revision**：读者不得看到半次扫描或作品与搜索索引不同 revision。
5. **规则是唯一业务解释入口**：Provider 名称、目录形状和 metadata 差异不得散落为业务代码分支。
6. **服务器拥有协议语义**：排序、分页、授权和有效字段解析由后端定义，客户端不得自行推断另一套规则。
7. **权限按 capability 判定**：角色仅是预设包；UI 只消费后端返回的 effective capabilities。
8. **跨平台核心不依赖桌面或 Windows API**：系统能力必须经适配层；壳可替换。
9. **错误必须结构化且可恢复**：空结果、离线、权限不足、规则错误、游标过期和内部失败不得混为同一种状态。
10. **不伪造证据**：仅交叉编译不等于运行支持，合成基准不等于真实硬件 SLA。

## 产品完成标准

Gallery 的成功标准不是“界面可打开”，而是：媒体根零写入；不可重建数据可备份恢复；Catalog 可删除重建；扫描期间查询一致；崩溃后任务状态可解释；API/事件/排序/游标均有版本；各客户端获得相同授权结果；支持矩阵中的平台均通过真实运行门禁。
