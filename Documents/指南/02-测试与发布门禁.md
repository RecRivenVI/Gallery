# 测试与发布门禁

> 类型：实施指南。Schema/API 冻结、平台支持和 v1 发布验收以本文为唯一权威来源。

## 测试原则

- 领域不变量和公开契约先于实现细节；
- 合成测试用于可重复和极端边界，真实只读样本用于验证规则覆盖，二者不得互相冒充；
- 平台支持必须在目标系统实际运行；交叉编译只能满足“可构建”；
- 性能报告必须记录样本、硬件、存储、版本、冷/热状态和分位数；
- 生产媒体根只允许只读验证，并用前后清单/guard 证明零变化；
- 高风险真实全库扫描、批量派生和维护任务不作为普通 CI。

## 持续测试层次

| 层 | 每次提交 | 定期/发布前 |
| --- | --- | --- |

| 数据 | Schema 升级、事务、备份恢复、revision、Saga、游标 | 强杀、磁盘满、损坏、GC/VACUUM |
| 契约 | OpenAPI、错误 code、WS 信封、权限矩阵 | 多客户端对拍、旧客户端拒绝/升级路径 |
| 集成 | 合成 Source 垂直切片、媒体 Range、规则保存 | 百万/千万基准、真实只读样本 |
| UI | mock API、键盘、焦点、能力显隐、断线恢复 | 主流浏览器、移动 viewport、壳 WebView |
| 平台 | Windows 核心 smoke | Windows 完整、Linux ext4、macOS、Docker、SMB 按支持目标 |

## Walking Skeleton 开工约束

Walking Skeleton 可以在完整 Schema/API 冻结前开工，但切片内已使用的基础契约必须是正式模型：两库迁移框架、Canonical/Source ID、SourceRuleBinding、首次完整 Blob hash、`query_publication_id`、OpenAPI DTO/错误信封和 capability 检查。验收应主动拒绝内存替代库、路径主键、临时 DTO、匿名管理员和客户端直连数据库。尚未实现 FTS、Overlay 写入、快照分页和多强杀点不构成开工阻塞，它们由 Architecture Proof Slice 冻结前补齐。

## 门禁分级

### Correctness Gate

所有正式支持环境必须满足，不能用“设备较慢”降级：

- Source 媒体根零写入，AppDirs 与 Source 不重叠；
- 同一 AppDirs 同时只有一个进程取得写所有权：第二个实例在打开或迁移数据库、建立监听、改写 descriptor 前即失败，锁在进程退出或强杀后释放，遗留锁文件不永久阻止启动，不同 AppDirs 可并行；
- 长任务经有界调度器执行：并发上限、资源类别隔离、取消与 graceful shutdown 生效，同一 Job 不并发执行两次，未完成 Job 由启动 reconciliation 收敛，不丢失 publication 一致性；
- 对外查询只能通过合法 `query_publication_id` 选择快照，不接受裸 revision 组合，也不混合 Catalog、Overlay projection、SortKey 和 FTS 代次；
- 取消、崩溃、磁盘满和跨库 reconciliation 后旧数据仍一致且可解释；
- Catalog 删除重建后 Canonical、Binding、Overlay 和固定 Blob 稳定引用不丢失；
- SourceWork 拆分/合并按稳定来源证据检测，无法安全自动处理时经 Binding issue 阻塞该 Source publication，人工决策以 control.db pre-seed Binding 表达；撤回仅适用于尚未被扫描消费的决策，消费后返回 `CONFLICT` 且决策、issue、Binding 不部分修改，已生效结构变化需新的人工决策或未来补偿流程。Catalog 全量重建后决策恢复、Canonical ID 不漂移、被否定候选不复现，其他 Source 不受影响；
- capability、资源范围、Session/Token/WS 吊销不可绕过；
- 路径解析不能越界，句柄复核和离线状态不能退化为不受控读取；
- 所有失败使用稳定 code，不泄露绝对路径、secret、内部 row ID 或私密 metadata。

### Reference Performance Gate

性能阈值只对明确登记的官方参考硬件成立。每次结果必须同时记录 CPU、内存、存储型号/介质、操作系统、SQLite/搜索库版本、冷/热缓存、并发数、样本规模、运行次数和分位数；缺少任一项的旧数字只能作为方向性证据，不能宣称通过正式门禁。

- 百万 WorkProjection、每个投影两条关系、完整 SourceMedia/ContentBlob/FileLocation/FTS，1%/10%/50% 变化的 publication P95 <250 ms；
- 搜索 P95 必须按查询类别、limit、候选量和是否包含原文复核分别设阈值，不能以单个选择性查询代表全部搜索；
- staging 构建、Overlay 重投影、GC、VACUUM、峰值临时空间和完整哈希吞吐必须各有预算；当前证据仅提供基线，正式阈值在参考硬件复跑后冻结。

### Degradation Gate

HDD、NAS 或低性能设备允许超过参考机性能阈值，但必须保证：

- 旧 `query_publication_id` 在构建、GC 和维护期间仍可浏览；
- 长任务有单调进度、可取消且不长时间持有 publication 锁；
- 空间预检失败时在 publication 前终止，保留旧数据；构建中磁盘满也不得破坏活动 revision；
- 维护窗口、积压和预计空间对用户可见；
- 性能退化不能改变快照、授权、路径安全、错误 code 或崩溃恢复语义。

## 领域 Schema 最终冻结门禁

> 分层说明：阶段 1 Schema Freeze Gate 已冻结**核心领域身份与唯一约束**（见 [领域模型与数据所有权](../规范/03-领域模型与数据所有权.md#schema-冻结分层与仍需确定项) 与 `schema_freeze` 表），其中含 Binding 的 `(source_id, source_key) WHERE status='active'` 唯一与 `(work_id, ordinal)` 唯一。本节是**最终物理唯一约束**冻结门禁，仍未通过；它进一步要求文件身份、超大文件持久哈希、SMB/FileID 降级等证据，未通过前不得冻结 FileLocation 等依赖文件身份的最终唯一约束，但不重开已冻结的核心领域身份。

必须完成：

- SourceWork/SourceCreator/SourceMedia 稳定引用冲突、重命名、跨 Source 和消失重现；
- CanonicalCreator/CanonicalWork 合并、撤销和误绑定人工修复；
- CanonicalMedia/ContentBlob/FileLocation 在复制、硬链接、symlink、跨卷移动、内容替换和采样碰撞下的语义；
- 新 ContentBlob 首次完整 SHA-256 在 publication 前完成；超大文件/网络盘具备持久 Job、字节进度、取消、退避、身份复核和失败重试，未完成 digest 永不发布；
- 完整性校验能发现“同大小、恢复时间戳、采样块相同而中间字节变化”的替换；普通扫描的非承诺边界在 API/UI 中明确；
- DerivedAsset 登记删除后，仅完整 key + 受校验 manifest 可接管缓存；transform/参数/Overlay 版本失效、orphan GC、singleflight、原子发布和读取 lease 均有并发测试；
- SMB 重挂载、inode/FileID 复用和无稳定文件身份时的降级；
- Catalog 删除重建后 Binding、Override、Favorite、Progress 和 CustomCover 不丢；
- `control.db` 中不存在指向 Catalog revision 内部 Source/Blob/Location row ID 的永久外键；固定内容版本以算法版本 + digest 重命中；
- control 备份/恢复和版本升级失败回滚；
- 明确 Canonical 删除、Blob hash version 和多值字段解析策略。

本门禁通过前，除阶段 1 已冻结的核心领域身份外，只能冻结方向和主外键语义，不能冻结依赖文件身份的最终物理唯一约束（尤其是 FileLocation 在 SMB/inode/无 FileID 下的约束）。

## Catalog 和任务门禁

- Reference Performance Gate 按上节参考环境执行 publication、构建、Overlay 重投影、GC/VACUUM、空间和哈希指标；不得把 `<250 ms` 解释为慢设备的正确性要求；
- 构建期间持续翻页，所有响应固定在原 `query_publication_id`；裸 revision 不能选择或组合快照；
- 构建取消、扫描进程强杀、发布事务强杀和发布后 control 未更新均恢复正确；
- Source*、ContentBlob/FileLocation、WorkProjection/CreatorProjection/MediaProjection、SortKey、关系和 FTS 使用合法的同一 revision 元组，无混代次；
- `published + job running` 自动修正 completed；`job completed + no publication` 进入 needs_repair；
- Overlay control 写同步、投影异步；按当前查询生成 dependency set，覆盖读己之写水位屏障、连续写合并、superseded 候选、失败重试和前端 saved/pending/failed/published 状态；
- GC 不删除活动/租约 `query_publication_id` 引用的 revision，过期游标得到 `CURSOR_EXPIRED`；
- SSD 与 HDD 分别测量构建、GC、VACUUM 和磁盘峰值；空间不足在发布前失败并保留旧 revision 元组。

## 规则门禁

- RulePackage/Draft/RuleVersion/ParameterSet/TestCase/Audit 生命周期在 `control.db` 闭环；invalid 草稿可保存且验证失败不覆盖原输入，发布版本不可变，弃用和回滚均有 revision/Impact/审计；
- `00017_rules_lifecycle` forward-only migration 通过旧数据升级、checksum、control 备份恢复和 Catalog 删除重建回归；规则事实不写入 Source，缓存丢失可重新编译；
- 规范 JSON、Schema、表单和高级编辑器无损往返，包括未知扩展；
- JSON/YAML/TOML 等价输入得到同一 canonical JSON 和 semantic identity；重复键、YAML alias/multi-document、TOML 日期/浮点和数字精度边界有拒绝或等价黄金测试；
- Gallery 规范化阶段显式实例化 JSON Schema default；字符串规范化必须由 Schema 逐字段声明，regex、glob、路径、JSON Pointer、metadata 键和 external ID 逐 code point 保持；
- JSON 数字以任意精度 coefficient/exponent 解析和比较，不经过 `float64`；`1`/`1.0`/`1e0` 的精确等价及超限拒绝有黄金测试；
- `package_hash` 覆盖完整包并随 tests/说明改变；`semantic_hash` 是 RuleVersion 身份且排除 tests-only 内容；`rule_ir_hash` 对实际 compiler/Profile/primitive registry/规范化版本及计划特化参数敏感；
- tests-only 修改只产生新 package artifact，不触发新 RuleVersion 或重扫；语义字段同时变化时必须产生新 `semantic_hash`；
- 同一校验器覆盖编辑、保存、启动和 CLI；
- CEL 超长度、AST、数组、输入、regex、cost 和超时均产生稳定错误；
- 无文件、网络、进程、时钟、随机、反射或未注册 host 函数；
- Dry Run/Explain 默认不泄露 metadata 值；
- RuleImpact 能区分 `NO_ACTION`、`REPROJECT`、`RESCAN_PARTIAL`、`RESCAN_FULL`、`BINDING_REVIEW` 和无效输入，包含原因码、hash、实体类型、Source 范围、人工确认和任务估计；
- SourceRuleBinding 的 active/条件/priority/ID 生效选择、同 priority 冲突、暂停、弃用规则、参数 override 和历史 Job 快照有测试；
- RulePackage 的 diff 能区分 display/query/identity/执行/参数/extension 变化；回滚失败不改变 current version；
- 内置规则对三类脱敏合成目录、缺失字段、非媒体作品、条件数组和媒体顺序有黄金断言，并通过示例列表/测试 API；
- 阶段 2 新增草稿、参数、HTTP API、Job 快照和恢复测试在 Windows 与 Linux/WSL `-race` 下运行，OpenAPI 生成物与源码一致。

## 查询和 API 冻结门禁

- 100 万和 1000 万样本覆盖双/多字中日文、拉丁、全角半角、数字、文件名中缀和结构化过滤；
- 候选召回和原文复核准确率为 1.0；无上限宽查询被拒绝或分页；
- 排序黄金语料覆盖自然数字、大小写、NFKC、emoji/grapheme、null、升降序和独立 ID tie-break；
- Go 生成的排序键由所有客户端按服务端顺序显示，不做本地重排；
- 游标覆盖 query、sort version、`query_publication_id`、auth scope、GC、过期和篡改，并证明客户端不能组合裸 revision；
- 冻结宽搜索 total、排名、高亮偏移，以及按当前查询 dependency 决定 snapshot predicate/live presentation 的 OpenAPI 表达；
- OpenAPI 枚举资源、错误 code、幂等/并发条件、capability 和所有分页字段；
- WebSocket 定义 snapshot、sequence、重连、Session 吊销和 HTTP 恢复；
- 空结果、离线、权限不足、校验错误、冲突和内部失败有独立 UI/客户端行为。

## 安全门禁

- Personal 匿名非管理员、一次性配对不可复用、Host/Origin/CSRF/Fetch Metadata 拒绝路径；
- Cookie 属性、空闲/绝对过期、多标签、Session/API Token/WS 吊销；
- LAN Owner 初始化、Argon2id 参数、角色预设与每资源 capability 矩阵；
- available/effective 交集在 API、Web、CLI 一致；
- 分享 scope/过期/吊销和固定 Blob 语义；
- 路径 traversal、编码变体、symlink/junction/reparse 和 TOCTOU 句柄复核；
- metadata 深度/大小、CEL 限额、恶意媒体/压缩炸弹、ffmpeg 超时/内存/输出限制；
- 日志、诊断、错误和审计不含 secret、绝对媒体路径或私密 metadata；
- Remote 模式保持不可启用，除非独立安全门禁全部完成。

## 平台与文件系统门禁

| 场景 | 必测行为 |
| --- | --- |
| Windows NTFS | 大小写、长路径、FileID、硬链接、symlink、junction/reparse、锁文件、签名发行 |
| Linux ext4 | 大小写、dev+inode、inotify 溢出、权限、非 root、systemd/无桌面 |
| macOS APFS | NFC/NFD、大小写变体、FSEvents、Keychain、签名/公证、AppDirs |
| Docker | 只读媒体卷、独立可写 AppDirs、非 root、信号关闭、固定 ffmpeg |
| SMB/NAS/UNC | 断线、重连、权限变化、FileID 变化、延迟、离线保留、不误删 |
| HDD | staging 构建、哈希、GC/VACUUM、取消响应和维护窗口 |

只有目标平台达到 [跨平台与客户端](../规范/09-跨平台与客户端.md) 定义的级别，发布说明才可宣称相应支持。

## Web 和桌面门禁

- Web 在无壳普通浏览器完成 v1 业务闭环；
- 键盘、焦点陷阱、ARIA、对比度、reduced motion、触控目标和媒体手势；
- 路由、搜索、分页和 WebSocket 的迟到响应不能覆盖新状态；
- UI 只消费 Schema、结构化 code 和 effective capabilities；
- 壳只能经 API/descriptor 工作，后端被独立启动时全部功能仍可用；
- 若采用 Wails，必须完成托盘、自启、目录选择、CredentialStore、崩溃退避、更新和签名，并与 Tauri 形成维护/包体对照。

## v1 发布检查单

1. 所有接受 ADR 的契约测试通过；有条件 ADR 在发布说明中明确状态。
2. `control.db` 备份、恢复和失败回滚演练通过；Catalog 删除重建不丢用户事实。
3. 支持矩阵逐项有实际运行证据；未通过平台不写“支持”。
4. 媒体根前后 guard 一致，AppDirs 与 Source 无重叠。
5. OpenAPI、WebSocket、规则 Schema、排序协议和数据 schema 都有版本。
6. 依赖、外部工具、许可证、SBOM、签名和漏洞检查完成。
7. 升级/降级边界、备份要求、重建影响和失败恢复写入发行说明。
8. 未解决门禁有明确 owner、影响和禁用策略，没有“先上线再观察”的静默例外。
9. Correctness、Reference Performance 和 Degradation 三类结果分别报告，不用参考机数字替代慢设备退化验证，也不用慢设备豁免正确性。
