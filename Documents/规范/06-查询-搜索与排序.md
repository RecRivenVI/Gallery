# 查询、搜索与排序

> 类型：规范。有效字段、过滤、搜索、排序和分页语义以本文为唯一权威来源。

## 查询对象

所有列表查询都面向服务端已发布的 `query_publication_id`，它不可拆分地引用一个合法 `(catalog_revision, overlay_projection_revision)`，并在授权后的 Library/Source/CanonicalWork 范围内执行。客户端不能分别指定两类 revision；查询层从 Source-derived 事实和 `control.db` Overlay 生成有效字段，但不得把查询投影当作新的权威事实。

客户端提交查询意图，服务器负责字段解析、过滤、搜索、排序、分页和 capability 裁剪。不同客户端不得自行复刻排序键或在取回一页后重新排序。

## 查询快照的组成

`catalog_revision` 冻结 SourceWork/SourceCreator/SourceMedia、ContentBlob/FileLocation 和来源关系；`overlay_projection_revision` 冻结基于一个确定 Catalog revision 生成的 WorkProjection/CreatorProjection/MediaProjection、有效字段、可见性、SortKey、结构化关系与 FTS。服务端为每个合法组合创建 `query_publication_id`；查询、租约和 GC 只通过该 ID 选择组合。

Overlay 不按字段永久划分一致性类别。query planner 必须根据当前查询生成 `overlay_dependency_set`：

- 某个 Overlay 值参与当前查询的过滤、排序、搜索、可见性或集合判断时，其判定值来自 `query_publication_id` 对应的 Overlay projection，并在游标租约内固定；
- 同一字段在另一个不依赖它的查询中，可以作为响应时从 `control.db` 读取的实时 `userState`，但不能改变该页集合与顺序；
- Schema 只声明字段支持 filter/sort/search/set/visibility 中的哪些能力，不声明它永远属于某一种一致性类别；OpenAPI 按查询用途区分 snapshot predicate 与 live presentation。

同一事实可以有“快照判定值”和“实时附加值”：例如按 Favorite 过滤时，成员资格按 publication 快照计算，而响应的 `userState.favorite` 可以显示当前值。两者不一致表示游标仍在旧 publication 中，不得用实时值在客户端删除、插入或重排当前页；刷新第一页后进入新 publication。

## v1 结构化查询

v1 至少支持：

- Library、Source、Provider 和 Origin 范围；
- CanonicalCreator、WorkCreator role、标签、媒体种类、可用/离线状态；
- 发布时间/发现时间范围、R-18/AI 等来自规则的规范字段；
- Favorite、HiddenState、Collection 和 ReadingProgress 等授权用户投影；
- 标题/CanonicalCreator/标签/文件名搜索；
- 已注册的排序字段及升降方向。

过滤字段必须在 OpenAPI/Schema 中明确类型、空值、组合方式和是否可排序。未知过滤字段返回结构化 4xx，不得静默忽略。

## 搜索索引 v1

v1 使用 SQLite FTS5，并明确分离候选索引和权威复核文本：

```text
normalized_original_text  # NFKC + Unicode case fold，连续子串复核和高亮来源
cjk_bigram_token_text     # 连续 CJK bigram 候选
filename_trigram_text     # 拉丁、数字和文件名连续 trigram 候选
structured_relations      # 标签、Provider、CanonicalCreator 等精确过滤
```

候选 token 使用 AND 查询，命中后必须在规范化原文中验证连续性和顺序。token 不能单独证明命中，也不能直接给出高亮偏移。精确标签和 ID 使用关系表，不依赖全文分词。

## v1 搜索承诺

| 能力 | v1 语义 |
| --- | --- |
| 至少两个 CJK 字符的连续搜索 | 支持，bigram 候选 + 原文复核 |
| 拉丁/数字和文件名中缀 | 支持，trigram 候选 + 原文复核 |
| 大小写、全角半角 | NFKC + Unicode case fold 后匹配 |
| 标签/Provider/CanonicalCreator 精确过滤 | 使用结构化关系 |
| 单字符 CJK | 无强过滤时返回 `QUERY_TOO_SHORT`；不得静默全库扫描 |
| 拼音、日文汉字读音 | 仅搜索显式 `sort_reading`/语言字段；不自动猜测 |
| 模糊纠错和语义搜索 | 延后 |
| 高亮 | 从复核后的规范化原文映射；正式偏移算法须过黄金集 |

宽查询必须分页并尽早复核，不能把所有候选搬入应用层。无上限 exact count 不是交互路径；精确总数、估算或省略的最终 API 表达仍是 Schema 冻结门禁。

## 排序协议 v1

```text
sort_protocol_version = 1
text_key = natural_chunks(unicode_case_fold(NFKC(text)))
nulls = last
SQL = ORDER BY sort_key, work_id
```

- 数字块按去前导零后的有效长度和值排序，再以原数字长度确定性打破等值；
- 非数字块使用稳定 Unicode/UTF-8 回退，不依赖系统 locale；
- `work_id` 是独立 SQL tie-break，不拼入 sort key；
- null 永远排在非 null 之后，升降序的具体编码必须在黄金语料中固定；
- 拼音和日文读音仅来自 metadata、用户输入或版本化语言模块提供的 `sort_reading`；缺失时回退基础协议；
- 协议升级必须更改 version，使旧游标过期。

时间排序必须先解析为明确 instant，并保留原始时间和解析规则版本。不得按未规范化时间字符串或 OS 本地时区排序。

## 快照分页

Gallery API v1 对有序列表承诺 revision 快照一致。签名游标至少绑定：

```text
query_fingerprint
sort_protocol_version
query_publication_id
authorization_scope_hash
last_sort_key
last_canonical_work_id
issued_at
lease_id / expires_at
```

后续页使用与 SQL 相同的 keyset 元组；示意为 `(sort_key, canonical_work_id) > (?, ?)`，实际比较符号随方向确定。`query_publication_id` 是服务端签名游标内唯一可选择快照的句柄，不是 SQLite row ID；裸 `catalog_revision`/`overlay_projection_revision` 即使出现在诊断响应中也不能作为查询参数。

| 变化 | 行为 |
| --- | --- |
| 新 query publication 发布 | 租约内继续旧 `query_publication_id`；新查询才进入新 publication |
| 当前 revision 中排序字段变化 | 不会发生；revision 不可变 |
| 查询/过滤改变 | `CURSOR_EXPIRED` |
| 授权范围、Session 或 Library 状态改变 | `CURSOR_EXPIRED` |
| 排序协议升级 | `CURSOR_EXPIRED` |
| query publication 已 GC、内部配对失效或租约过期 | `CURSOR_EXPIRED` |
| 游标签名/结构无效 | `CURSOR_INVALID`，不泄露内部值 |

服务器不为长期离线游标无限保留 revision。客户端收到 `CURSOR_EXPIRED` 后必须从第一页刷新，不得猜测游标内部字段。

## 一致性和缓存

- 响应必须带查询使用的 `query_publication_id` 和排序协议版本；可附带两类 revision 作为只读诊断信息；
- ETag/缓存键包含查询指纹、`query_publication_id` 和授权范围；带实时 `userState` 的私有响应还必须包含用户状态版本或禁止共享缓存；
- 搜索文档、有效字段、结构化关系和 WorkProjection 必须与同一 revision 元组发布；
- 当前查询依赖的 Overlay 只通过新的 query publication 生效，不允许一页内混用新旧有效字段；未进入本次 `overlay_dependency_set` 的实时字段不得反向参与集合、排序、搜索、过滤或可见性；
- 客户端本地筛选只能用于纯展示，不得改变服务端声明的总集合语义。

## Schema 冻结前门禁

- 正式排名、字段权重和高亮黄金集；
- 宽查询的总数返回策略和预算；
- emoji/grapheme、组合字符和 null 降序的完整黄金语料；
- 游标租约、最大 revision 保留和空间上限；
- `query_publication_id` 不能组合裸 revision、不能跨授权复用且随 GC/租约正确失效的契约测试；
- Overlay projection 构建失败/积压、连续写水位、读己之写屏障、实时字段标注和旧 publication GC 的契约测试。
